## 프로젝트 매니저

| 갭 | 보완 방법 |
|----|-----------|
| 프로젝트 기간(1주일 포트폴리오)이 명시되지 않음 → 평가자가 기간 대비 완성도를 판단할 수 없음 | 섹션 1에 "개발 기간: 1주일 (수업 평가용 포트폴리오)" 한 줄 추가 |
| 핵심 가치 제안(Value Proposition)이 기획 배경 + 기대 효과로 분산되어 PPT 첫 슬라이드에 쓸 한 문장이 없음 | "영어 뉴스 장벽 없이, 매일 아침 내 관심 종목 브리핑을 30초 만에" 같은 훅 문장을 섹션 2 앞에 추가 |
| "왜 지금인가" 타이밍 근거가 정성적 설명만 있음 (서학개미 급증 수치 등 데이터 없음) | 기획 배경에 "2024~2025년 서학개미 계좌 수 급증" 등 공개 지표 한 줄 보강 (없으면 "급증" 표현 유지) |

## AI 기능 엔지니어

| 갭 | 보완 방법 |
|----|-----------|
| CLOVA Studio 실패 시 fallback이 없음 — 번역 실패는 영문 원문 진행이 적혀 있지만 요약 실패 시 동작이 문서에 없음 | 섹션 4-2에 "에러 처리: 요약 실패 시 해당 종목 item 생성 건너뜀(부분 성공 허용)" 추가 |
| 감성 분석 결과가 화면에서 어떻게 시각화되는지 연결 설명 없음 | 섹션 4-2 출력 항목에 "→ 화면에서 감성 배지(초록/빨강/회색) 색상으로 표시" 한 줄 추가 |
| 실제 AI 출력 샘플이 없음 — NVDA 같은 반도체 종목 요약·감성 결과 예시가 있으면 발표 설득력 증가 | 섹션 4-2에 실제 또는 예시 출력 JSON 한 블록 추가 (예: NVDA 요약문 + "positive") |
| 기사 수집 건수(최대 5건)와 요약 품질의 관계가 설명되지 않음 | 섹션 4-2 입력 항목에 "기사 최대 5건을 묶어 종합 요약 → 단일 기사보다 맥락 풍부" 부연 추가 |

## Ncloud 설계자

| 갭 | 보완 방법 |
|----|-----------|
| NCP API Gateway 인증 공유 구조가 설명되지 않음 (Papago·CLOVA Voice가 같은 키를 쓰는 이유) | 섹션 5 하단에 "NCP API Gateway 공통 인증: NCP_APIGW_KEY_ID/KEY 하나로 Papago·CLOVA Voice 모두 인증" 한 줄 추가 |
| Object Storage mp3 공개 접근 정책(public-read ACL) 언급 없음 | 섹션 5 Object Storage 행의 선택 이유에 "public-read ACL로 별도 서명 없이 재생 URL 직접 서빙" 추가 |
| Finnhub을 외부 서비스로 쓰는 이유가 빠짐 (NCP에 동등한 금융뉴스 API 없음) | 외부 서비스 표에 Finnhub 행의 선택 이유 컬럼 추가: "NCP에 영어 금융뉴스 수집 서비스 없음, 무료 티어 충분" |
| 서비스 연결 아키텍처 다이어그램이 없음 | PPT 슬라이드용으로 섹션 3의 파이프라인 텍스트를 박스 다이어그램으로 시각화 권장 |

## 디자인 팀장

| 갭 | 보완 방법 |
|----|-----------|
| 실제 화면 스크린샷이 없음 — PPT에 넣을 이미지 자료 안내가 없음 | 섹션 6 하단에 "PPT 삽입용 스크린샷 필요 화면" 체크리스트 추가 (홈·상세·관심종목 편집 3장 최소) |
| 로딩 상태(온디맨드 검색 중 스피너) 화면이 데모 목록에 없음 | 섹션 6 표에 "검색 로딩" 행 추가: `POST /search` 제출 후 스피너·버튼 비활성 상태 캡처 필요 |
| 감성 배지 색상 규칙이 PPT 자료로 시각화되지 않음 | 섹션 6 또는 섹션 4-2에 색상 표 추가: positive `#00c471` / negative `#f04452` / neutral `#8c919e` |
| 모바일 반응형 여부가 언급되지 않음 | 섹션 6 데모 추천 흐름에 "모바일 브라우저에서도 단일 컬럼 레이아웃 동작" 한 줄 추가 |

## 기능 검증 팀장

| 갭 | 보완 방법 |
|----|-----------|
| pytest 단위 테스트 결과가 없음 — HTTP 엔드포인트 QA만 있고 파이프라인 내부 단위 테스트 언급 없음 | 섹션 7에 `pytest -q` 실행 결과(통과/실패 수) 한 줄 추가, 또는 "외부 API 전체 mock, 단위 테스트 포함" 명시 |
| 실제 API 응답 JSON 샘플이 없음 | 섹션 7 QA 표 하단에 `/api/quotes` 또는 `/search` 실제 응답 JSON 일부 추가 |
| 자동화 테스트와 수동 테스트 구분이 불명확 | QA 표 제목을 "QA 자동화 테스트(requests 라이브러리 기반)" 또는 "QA 수동 HTTP 확인"으로 명확히 수정 |
| TTS 실제 생성 증거(mp3 URL 또는 재생 확인)가 없음 | 섹션 7에 "Object Storage mp3 업로드 확인: `audio/{date}/AAPL.mp3` 실제 재생 URL 확보" 추가 |

## 보안 팀장

| 갭 | 보완 방법 |
|----|-----------|
| 실서비스 URL에 서버 IP가 노출되어 있음 — 이 파일이 git 커밋되면 IP 공개됨 | 섹션 1의 실서비스 URL에서 IP 제거 또는 `http://<서버-IP>:8000` 형태로 마스킹, 파일을 `.gitignore`에 추가 |
| 화면 녹화·공유 시 브라우저 주소창에 IP 노출되는 시나리오가 데모 주의사항에 빠짐 | 섹션 9 데모 주의사항에 "화면 녹화 전 브라우저 주소창 숨기거나 블러 처리" 명시 |
| 데모 계정 자격증명 관리(테스트 계정 비밀번호 노출) 주의사항 없음 | 섹션 9에 "데모용 테스트 계정 비밀번호를 공개 채팅/슬라이드에 적지 않기" 추가 |

## 금융 도메인 팀장

| 갭 | 보완 방법 |
|----|-----------|
| "정보 격차"가 실제 투자자 불이익으로 이어진다는 맥락이 약함 — "불편하다"에 그침 | 해결하려는 문제 첫 줄을 "영어 뉴스를 빠르게 소화하지 못하면 정보 후행 → 고점 매수·저점 매도 위험"으로 강화 |
| 감성 분석(positive/neutral/negative)이 투자 판단에 주는 가치가 설명되지 않음 | 섹션 4-2 하단에 "감성 배지는 뉴스 맥락의 빠른 시각적 파악을 돕는 참고 지표(투자 조언 아님)" 한 줄 추가 |
| 면책 문구("투자 조언 아님")가 PPT 자료에 전혀 언급되지 않음 — 실제 서비스 푸터에는 있음 | 섹션 2 기대 효과 하단 또는 섹션 9에 "푸터 고정 면책: '본 서비스는 투자 조언을 제공하지 않습니다'" 추가 |
| 고정 WATCHLIST 종목(NVDA·MSFT 등) 선택 이유가 AI·반도체 호황 맥락과 연결되지 않음 | 섹션 2 대상 사용자 표 하단에 "고정 종목은 AI·반도체 호황의 핵심 종목(NVDA·MSFT·AAPL 등)으로 구성" 한 줄 추가 |
| 기대 효과가 편의성("30초", "출퇴근") 중심 — 정보 격차 해소라는 더 강한 가치 주장이 약함 | 기대 효과에 "언어 장벽 없이 국내 투자자도 동일한 정보에 동시 접근 가능" 항목 추가 |

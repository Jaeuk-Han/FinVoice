## 3. 장표 구성 — 핵심 메시지와 넣을 자료

### 01. 표지
- **핵심 메시지**: "영어 뉴스 장벽 없이, 매일 아침 내 관심 종목 브리핑을 30초 만에"
- **자료**: 서비스명(FinVoice), 부제(해외 금융뉴스 AI 음성 브리핑 서비스), 개발 기간
- **디자인**: 다크 배경, 브랜드 블루(`#3182f6`) 강조, 군더더기 없이

---

### 02. 기획 배경
- **핵심 메시지**: AI 열풍 → 반도체 호황 → 서학개미 급증 → 영어 뉴스 장벽
- **자료**:
  - "AI 열풍이 엔비디아(NVDA)·마이크론 등 반도체 주가 호황을 이끌면서 국내 투자자의 미국 주식 관심이 급증"
  - 흐름: `AI 열풍 → 반도체 주가 호황 → 서학개미 증가 → 핵심 정보는 영어 → 정보 격차`
  - 키워드 강조: **정보 격차(Information Gap)**
- **디자인**: 단순 흐름 화살표 다이어그램. 텍스트 최소화.

---

### 03. 문제 정의 & 대상 사용자
- **핵심 메시지**: "영어 뉴스를 빠르게 소화하지 못하면 정보 후행 → 고점 매수·저점 매도 위험"
- **자료**:

  **문제**
  - 영어 뉴스를 빠르게 소화하지 못하면 정보가 뒤처져 고점 매수·저점 매도 위험
  - 시간이 없거나 영어가 불편한 국내 투자자는 동일 정보에 동시 접근이 어려움
  - "오늘 내 관심 종목이 왜 올랐는지/내렸는지"를 빠르게 파악할 수단이 부족

  **대상 사용자**
  | 유형 | 설명 |
  |------|------|
  | 비로그인 | 고정 종목(AAPL·TSLA·NVDA·MSFT·AMZN·GOOGL) 브리핑 열람 |
  | 로그인 | 관심종목 직접 설정(최대 5개), 임의 종목 검색 |

  > 고정 종목은 AI·반도체 호황의 핵심 빅테크·반도체 종목으로 구성

- **디자인**: 왼쪽 문제 3개 / 오른쪽 사용자 표. 심플 2단 레이아웃.

---

### 04. 솔루션: 핵심 파이프라인
- **핵심 메시지**: "4단계 AI 파이프라인으로 영어 뉴스를 음성 브리핑으로"
- **자료**: 박스 + 화살표 다이어그램

  ```
  Finnhub API          NCP Papago           NCP CLOVA Studio     NCP CLOVA Voice
  [영어 뉴스 수집]  →  [영→한 번역]    →   [요약 + 감성 분석] →  [TTS mp3 생성]
                                                                        ↓
                                                              NCP Object Storage
                                                              [mp3 저장·서빙]
                                                                        ↓
                                                              웹 화면
                                                       [요약 + 음성 플레이어 + 출처]
  ```

  **브리핑 생성 진입점 3가지 (동일 파이프라인 공유)**
  | 진입점 | 시점 |
  |--------|------|
  | cron 배치 | 매일 오전 7시 자동 |
  | 서비스 기동 시 | 서버 재시작 직후 (오늘 브리핑 없을 때만) |
  | 온디맨드 검색 | 사용자가 종목 검색 시 즉석 실행 |

- **디자인**: 파이프라인 박스 다이어그램 중심. NCP 로고 또는 색상 구분.

---

### 05. AI 서비스 상세
- **핵심 메시지**: "3개 NCP AI 서비스가 번역→요약→음성 각 단계를 담당"
- **자료**: 3개 카드

  **Papago NMT (번역)**
  - 입력: 영어 기사 제목 + 본문 발췌 (최대 4,500자/요청)
  - 출력: 한국어 번역문
  - 실패 시: 영어 원문으로 요약 단계 진행 (부분 성공 허용)

  **CLOVA Studio HCX-003 (요약 + 감성 분석)**
  - 입력: 종목명 + 번역된 기사 최대 5건
  - 출력: 한국어 요약 3~5문장 + positive / neutral / negative
  - 출력 → 화면에서 감성 배지(초록/빨강/회색)로 시각화
  - 실패 시: 해당 종목 건너뜀 (전체 배치 중단 없음)

  **CLOVA Voice (TTS)**
  - 입력: 한국어 요약문 (최대 1,900자)
  - 출력: mp3 → NCP Object Storage 업로드 → 공개 재생 URL
  - 실패 시: `audio_url=None`, 텍스트 요약만 표시 (비치명적)

- **디자인**: 3열 카드. 입력 → 출력 → 실패처리 순서로 정렬. 색상 구분.

---

### 06. CLOVA Studio: 프롬프트 전략 & 실제 출력
- **핵심 메시지**: "추측 금지·JSON 강제·감성 3분류 — 프롬프트 설계로 일관된 구조화 출력"
- **자료**:

  **프롬프트 전략**
  > "기사 내용에서만 근거하라. 추측하지 마라. 3~5문장 요약 + positive/neutral/negative 판단. JSON으로만 답하라."
  - 모델: HCX-003 / maxTokens: 500 / temperature: 0.3

  **실제 출력 예시 (NVDA)**
  ```json
  {
    "summary": "엔비디아가 AI 데이터센터 수요 급증으로 분기 매출 전망을 상향 조정했다. 블랙웰 GPU 공급 확대와 함께 주요 클라우드 기업들의 대규모 주문이 이어지고 있으며, 경쟁사 대비 AI 가속기 시장 점유율이 더욱 확대될 것으로 전망된다. 단기적으로는 수출 규제 리스크가 남아 있으나 전반적인 펀더멘털은 긍정적이다.",
    "sentiment": "positive"
  }
  ```

- **디자인**: 왼쪽 프롬프트 전략 / 오른쪽 코드 블록 출력 예시. 다크 코드 블록 스타일.

---

### 07. Ncloud 서비스 구성
- **핵심 메시지**: "6개 NCP 서비스를 단일 인증 체계로 통합 — 개발·운영 모두 NCP 안에서"
- **자료**:

  | 서비스 | 역할 | 선택 이유 |
  |--------|------|-----------|
  | Server (Ubuntu VM) | FastAPI 앱 + cron 배치 | systemd 자동 복구, 24/7 안정 운영 |
  | Cloud DB for MySQL | 브리핑·사용자·워치리스트 | 관리형 DB, 백업·이중화 부담 없음 |
  | Papago NMT | 영→한 번역 | NCP 통합 인증, 금융 번역 품질 |
  | CLOVA Studio (HCX-003) | 요약 + 감성 분석 | 한국어 금융 문맥 이해도 |
  | CLOVA Voice | TTS mp3 생성 | 자연스러운 한국어 음성 |
  | Object Storage | mp3 저장·서빙 | public-read ACL로 서명 없이 URL 서빙 |

  > NCP API Gateway 공통 인증: `NCP_APIGW_KEY_ID/KEY` 하나로 Papago·CLOVA Voice 공유

  **외부 서비스**: Finnhub (NCP에 영어 금융뉴스 수집 서비스 없음, 무료 티어 충분)

- **디자인**: NCP 서비스 박스 그룹 + 외부 서비스 구분. 연결 화살표로 데이터 흐름 표시.

### 08. 주요 화면 1 — 홈 & 브리핑 상세
- **핵심 메시지**: "미니멀 다크 UI — 정보가 먼저 보이는 설계"
- **자료**:
  - **스크린샷 삽입**: 홈 화면(티커바 + 날짜별 브리핑 목록)
  - **스크린샷 삽입**: 브리핑 상세(감성 배지 + 한국어 요약 + 음성 플레이어 + 출처 접이식)
  - 감성 배지 색상 규칙: positive `#00c471` / negative `#f04452` / neutral `#8c919e`
  - 실시간 주가 티커바: `▲/▼` + 등락률, 60초 서버 캐시
- **디자인**: 실제 스크린샷 중심. 텍스트 설명은 화살표 콜아웃으로 최소화.

### 10. 핵심 기능 구현
- **핵심 메시지**: "3가지 진입점이 동일 파이프라인을 공유 — 로직 중복 없는 설계"
- **자료**:

  **브리핑 생성 3진입점**
  ```
  cron (매일 07:00)  ─────┐
  서비스 기동 시      ─────┤──→ runner.process_symbol() ──→ DB 저장
  온디맨드 검색       ─────┘
  ```

  **자동 새로고침 폴링**
  - 관심종목 저장 후 `/?generating=1` 리다이렉트
  - `/api/today-item-count` 3초 간격 폴링 → 카운트 증가 시 자동 새로고침
  - 3분 후 타임아웃으로 무한 폴링 방지

  **실시간 주가 티커바**
  - 로그인: 개인 워치리스트 심볼 조회 / 비로그인: 고정 WATCHLIST
  - 서버 60초 캐시

- **디자인**: 다이어그램 + 핵심 구현 포인트 불릿. 코드 스니펫 1~2줄 첨부 가능.

### 11. QA 결과 & 오류 수정 기록
- **핵심 메시지**: "9/9 전체 통과, 8건의 실제 오류를 발견·해결"
- **자료**:

  **QA 테스트 결과 (자동화 HTTP, 2026-06-07)**
  | 엔드포인트 | 결과 |
  |------------|------|
  | GET / | PASS |
  | GET /login, /register | PASS |
  | GET /api/quotes | PASS — 6개 종목 주가 JSON |
  | POST /search (AAPL) | PASS — 요약·음성·출처 5건 |
  | GET /watchlist/edit (비로그인) | PASS — /login 303 리다이렉트 |
  | GET /briefing/1 (없는 ID) | PASS — 한국어 404, 스택트레이스 없음 |

  **주요 오류 수정 8건 (발췌)**
  | # | 오류 | 해결 |
  |---|------|------|
  | 1 | bcrypt 4.x passlib 비호환 | `bcrypt==3.2.2` 고정 |
  | 3 | 관심종목 주가 `—` 표시 | `/api/quotes` symbols 파라미터 처리 |
  | 4 | 워치리스트 저장 후 히스토리 미노출 | `insert_briefing` 선행 후 item 연결 |
  | 7 | cron 미등록 → 자동 배치 미실행 | paramiko로 cron 등록 + startup 이벤트 |
  | 8 | 티커바 오른쪽 카드 잘림 | `width:100%`, `flex:0 0 auto`, scroll-snap |

- **디자인**: 왼쪽 QA 표(초록 PASS) / 오른쪽 오류 수정 표. 심플 2단.

## 4. 디자인 톤 가이드

| 원칙 | 설명 |
|------|------|
| **실제 앱처럼** | 마케팅 슬라이드가 아니라 실제 동작하는 서비스 발표. 스크린샷과 코드 블록 적극 활용 |
| **다크 미니멀** | 배경 `#0d0e10`, 카드 `#16171b`, 강조 `#3182f6`. 화려한 그라데이션·애니메이션 금지 |
| **정보 밀도** | 슬라이드당 핵심 1개. 텍스트 블록보다 표·다이어그램·코드 블록 우선 |
| **감성 색상 유지** | positive `#00c471` / negative `#f04452` / neutral `#8c919e` — 앱과 동일하게 |
| **마케팅 금지** | 과장된 형용사, 불필요한 이모지, "혁신적인" 류 표현 사용하지 않음 |
| **폰트** | 제목: ChosunSg 또는 굵은 시스템 폰트 / 본문: system-ui / 숫자·코드: Inter |

## 5. 민감정보 금지 규칙

> 슬라이드·스크린샷·코드 블록 어디에도 아래 정보를 포함하지 않는다.

- 서버 IP 주소
- 서버 root 비밀번호 / SSH 접속 정보
- Finnhub / NCP API 키 (Papago·CLOVA·Object Storage 모두 포함)

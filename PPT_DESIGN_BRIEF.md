### 02. 기획 배경

**핵심 메시지**: "AI 열풍이 만든 정보 격차를 해소한다"

**포함할 자료**
- 흐름 다이어그램 (좌→우 화살표 4단계):
  ```
  AI 열풍  →  반도체 주가 호황  →  서학개미 급증  →  핵심 뉴스는 영어  →  정보 격차
  ```
- 각 단계 아래 한 줄 설명:
  - AI 열풍: NVDA·MSFT 등 빅테크 AI 투자 급증
  - 반도체 호황: NVDA 주가 2023~24년 300% 이상 상승
  - 서학개미 급증: 국내 해외주식 거래 급증
  - 정보 격차: 핵심 뉴스·실적·컨퍼런스콜 모두 영어

**디자인 지시**
- 화살표 흐름 다이어그램 중심, 텍스트 최소화
- 마지막 단계 "정보 격차"를 빨간 강조색으로 시각적 포인트

### 03. 문제 정의 & 대상 사용자

**핵심 메시지**: "영어 뉴스 못 읽으면 정보 후행 → 고점 매수·저점 매도"

**포함할 자료**

*왼쪽 — 해결하는 문제 3개*
1. 영어 뉴스를 빠르게 소화하지 못하면 정보가 뒤처져 고점 매수·저점 매도 위험
2. 시간이 없거나 영어가 불편한 투자자는 동일 정보에 동시 접근이 어려움
3. "오늘 내 종목이 왜 올랐는지" 빠르게 파악할 수단 부족

*오른쪽 — 대상 사용자 표*
| 유형 | 접근 권한 |
|------|----------|
| 비로그인 | 고정 종목 6개(AAPL·TSLA·NVDA·MSFT·AMZN·GOOGL) 브리핑 열람·검색 |
| 로그인 | 관심종목 직접 설정(최대 5개), Finnhub 임의 종목 검색 허용 |

> 고정 종목 = AI·반도체 호황의 핵심 미국 빅테크·반도체 종목

**디자인 지시**
- 2단 레이아웃: 왼쪽 문제 / 오른쪽 사용자 표
- 문제 3개는 번호 아이콘 또는 불릿

### 04. 솔루션: 4단계 AI 파이프라인

**핵심 메시지**: "영어 뉴스 → 한국어 음성 브리핑, 4단계 자동화"

**포함할 자료**

*파이프라인 박스 다이어그램*
```
[Finnhub]        [NCP Papago]      [NCP CLOVA Studio]    [NCP CLOVA Voice]
영어 뉴스 수집  →  영→한 번역  →  요약 + 감성 분석   →  TTS mp3 생성
                                                               ↓
                                                    [NCP Object Storage]
                                                    mp3 저장 → 공개 URL
                                                               ↓
                                                    [웹 화면]
                                                요약 + 음성 플레이어 + 출처
```

*진입점 3가지 표*
| 진입점 | 시점 | 대상 |
|--------|------|------|
| cron 배치 | 매일 오전 7시 자동 | 고정 6개 종목 |
| 서비스 기동 시 | 재시작 직후, 오늘 브리핑 없을 때만 | 고정 6개 종목 |
| 온디맨드 검색 | 사용자 검색 시 즉석 실행 | 요청 종목 1개 |

> 세 진입점 모두 동일한 `runner.process_symbol()` 호출 — 로직 중복 없음

*온디맨드 처리 중 상태 (별도 콜아웃 박스)*
```
검색 요청 → [파이프라인 실행 중... 수 초 소요] → 결과 표시
              로딩 스피너 표시 + 버튼 비활성화
              (중복 제출 방지)
```

**디자인 지시**
- 파이프라인을 슬라이드 상단 2/3 차지하는 메인 다이어그램으로
- 진입점 표는 하단 좌측
- 온디맨드 처리 중 콜아웃은 하단 우측 작은 박스로

### 05. AI 서비스 상세

**핵심 메시지**: "Papago 번역 → CLOVA Studio 요약 → CLOVA Voice 음성, 각 단계 실패도 전체를 멈추지 않는다"

**포함할 자료**

*3열 카드 레이아웃*

**카드 1 — NCP Papago NMT**
- 입력: 영어 기사 제목 + 본문 발췌 (최대 4,500자/요청)
- 출력: 한국어 번역문
- 실패 처리: 번역 실패 → 영어 원문으로 요약 진행 (부분 성공 허용)

**카드 2 — NCP CLOVA Studio HCX-003**
- 입력: 종목명 + 번역된 기사 최대 5건 (단일 기사보다 풍부한 맥락)
- 출력: 한국어 요약 3~5문장 + `positive / neutral / negative`
- 시각화: 감성 배지 — positive `#00c471` / negative `#f04452` / neutral `#8c919e`
- 실패 처리: 해당 종목 건너뜀, 전체 배치 중단 없음

**카드 3 — NCP CLOVA Voice**
- 입력: 한국어 요약문 (최대 1,900자, 화자: nara 여성)
- 출력: mp3 → NCP Object Storage 업로드 → 공개 재생 URL
- 실패 처리: `audio_url=None`, 텍스트 요약만 표시 (비치명적)

**디자인 지시**
- 3열 카드, 각 카드 내부에 입력/출력/실패처리 구분
- 카드 사이 화살표로 흐름 연결

### 06. CLOVA Studio: 프롬프트 전략 & 실제 출력

**핵심 메시지**: "추측 금지·JSON 강제·3분류 — 프롬프트 설계로 구조화된 출력 보장"

**포함할 자료**

*왼쪽 — 프롬프트 전략*
- "기사 내용에서만 근거하라. 추측하지 마라."
- "3~5문장 요약 + positive/neutral/negative 중 하나 판단"
- "반드시 JSON 형식으로만 답하라"
- 모델 파라미터: HCX-003 / maxTokens: 500 / temperature: 0.3
- 기사 5건 입력 → 단일 기사보다 맥락 풍부

*오른쪽 — 실제 출력 예시 (NVDA, 다크 코드 블록)*
```json
{
  "summary": "엔비디아가 AI 데이터센터 수요 급증으로 분기 매출 전망을 상향 조정했다. 블랙웰 GPU 공급 확대와 함께 주요 클라우드 기업들의 대규모 주문이 이어지고 있으며, 경쟁사 대비 AI 가속기 시장 점유율이 더욱 확대될 것으로 전망된다. 단기적으로는 수출 규제 리스크가 남아 있으나 전반적인 펀더멘털은 긍정적이다.",
  "sentiment": "positive"
}
```

- 오른쪽 하단: `positive` 배지 시각화 (`#00c471` 초록)

**디자인 지시**
- 왼쪽 프롬프트 전략 / 오른쪽 코드 블록 2단 분할
- 코드 블록은 다크 배경(`#0d0e10`) + 문법 강조
- 왼쪽 빈 공간 없도록 "기사 5건 → 단일 기사 대비 맥락 풍부" 설명 추가

### 07. Ncloud 서비스 구성

**핵심 메시지**: "6개 NCP 서비스를 단일 인증 체계로 통합 — 개발·운영 모두 NCP 안에서"

**포함할 자료**

*상단 — 서비스 연결 아키텍처 다이어그램*
```
[Server VM]
    │
    ├──→ [Cloud DB for MySQL]      briefing / user / watchlist 저장·조회
    │
    ├──→ [Papago NMT]              영→한 번역
    ├──→ [CLOVA Studio HCX-003]   요약 + 감성
    ├──→ [CLOVA Voice]            TTS mp3
    │         └──→ [Object Storage]  mp3 저장 → 공개 URL
    │
    └──→ [Finnhub API] (외부)     영어 뉴스 · 주가 · 종목 프로필
```

*하단 — 서비스 선택 이유 표*
| 서비스 | 역할 | 선택 이유 |
|--------|------|-----------|
| Server (Ubuntu VM) | FastAPI 앱 + cron 배치 | systemd 자동 복구, 24/7 안정 운영 |
| Cloud DB for MySQL | 브리핑·사용자·워치리스트 | 관리형 DB, 백업·이중화 부담 없음 |
| Papago NMT | 영→한 번역 | NCP 통합 인증, 금융 번역 품질 |
| CLOVA Studio (HCX-003) | 요약 + 감성 분석 | 한국어 금융 문맥 이해도 |
| CLOVA Voice | TTS mp3 생성 | 자연스러운 한국어, APIGW 키 공유 |
| Object Storage (S3 호환) | mp3 저장·서빙 | public-read ACL, 서명 없이 URL 서빙 |

> NCP API Gateway 공통 인증: `NCP_APIGW_KEY_ID/KEY` 하나로 Papago·CLOVA Voice 공유
> Finnhub: NCP에 영어 금융뉴스 수집 서비스 없음, 무료 티어로 충분

**디자인 지시**
- 아키텍처 다이어그램을 슬라이드 상단 절반에 배치 (이전 초안에서 누락된 항목)
- 하단에 선택 이유 표 (컴팩트하게)

### 08. 주요 화면 1 — 홈 & 브리핑 상세

**핵심 메시지**: "미니멀 다크 UI — 정보가 먼저 보이는 설계"

**포함할 자료**
- `[스크린샷: 홈 화면]` — 티커바(6개 종목 ▲/▼ + 등락률) + 날짜별 브리핑 목록 **3건 이상** 표시
- `[스크린샷: 브리핑 상세]` — 종목 카드(회사명·심볼·감성 배지·한국어 요약·음성 플레이어·출처 접이식)
- 감성 배지 색상 설명: positive `#00c471` / negative `#f04452` / neutral `#8c919e`
- 콜아웃: 실시간 주가 60초 서버 캐시, 출처 링크 항상 노출(접이식)

> 스크린샷은 실제 서버 화면 사용. 브라우저 주소창 크롭 또는 블러 처리.
> 브리핑 목록은 서비스가 며칠간 운영 중임을 보여주기 위해 3건 이상 표시.

**디자인 지시**
- 좌우 2단: 홈 화면 / 상세 화면
- 스크린샷이 없을 경우 이전 목업 유지 (단, 브리핑 목록 1건 → 3건 이상으로 수정)

### 09. 주요 화면 2 — 관심종목 편집 & 자동 생성 흐름

**핵심 메시지**: "로그인하면 내 종목, 내 브리핑 — 저장 즉시 백그라운드 자동 생성"

**포함할 자료**
- `[스크린샷: 관심종목 편집]` — 칩(chip) UI, 최대 5개 제한, 추가·삭제 버튼
- `[스크린샷: 로그인 화면]` — 이메일·비밀번호, 오류 인라인 표시
- 자동 생성 흐름 (4단계 화살표):
  ```
  ① 관심종목 저장 → ② 백그라운드 파이프라인 실행
       → ③ /api/today-item-count 3초 폴링
       → ④ 카운트 증가 감지 → 자동 새로고침
  ```
- 하단 안내: 모바일에서도 단일 컬럼 레이아웃 동작

**디자인 지시**
- 상단: 스크린샷 2개 나란히
- 하단: 자동 생성 4단계 흐름 다이어그램

### 10. 핵심 기능 구현

**핵심 메시지**: "3진입점 동일 파이프라인 공유 + 폴링 자동새로고침 + 실시간 티커바"

**포함할 자료**

*왼쪽 — 3진입점 공유 다이어그램*
```
cron (매일 07:00)  ──────┐
서비스 기동 시     ──────┤──→ runner.process_symbol()
온디맨드 검색      ──────┘         │
                              DB 저장 (캐시)
```
> 코드 한 줄 강조: `pipeline/runner.py: process_symbol(symbol, date)`
> "로직 중복 없음 — 세 진입점이 동일 함수 호출"

*오른쪽 — 두 가지 실시간 기능*

**폴링 자동 새로고침**
- 관심종목 저장 → `/?generating=1` 리다이렉트
- `/api/today-item-count` 3초 간격 폴링
- 카운트 증가 시 자동 새로고침 / 3분 후 타임아웃

**실시간 주가 티커바**
- `/api/quotes?symbols=...` 엔드포인트
- 서버 60초 인메모리 캐시 (Finnhub 한도 절약)
- 로그인: 개인 워치리스트 / 비로그인: 고정 WATCHLIST

**디자인 지시**
- 좌우 2단, 왼쪽 다이어그램 / 오른쪽 두 기능 설명
- 코드 한 줄 콜아웃으로 실제 구현 근거 표시

### 11b. 오류 수정 기록 ← 문제 해결 노력 15점

> ⚠️ 이전 초안에서 QA와 한 장에 합쳐졌던 슬라이드. **반드시 독립 슬라이드**로 분리.
> 이전 초안에는 8건 중 5건만 발췌했으나, 이번에는 **8건 전체 표시**.

**핵심 메시지**: "라이브러리 충돌부터 CSS 버그까지 — 8건의 실제 오류를 발견·해결"

**포함할 자료**

*오류 수정 8건 전체*
| # | 오류 현상 | 원인 | 해결 |
|---|-----------|------|------|
| 1 | `AttributeError: bcrypt has no attribute '__about__'` | bcrypt 4.x가 passlib 1.7.4와 비호환 | `requirements.txt`에 `bcrypt==3.2.2` 버전 고정 |
| 2 | 예외 핸들러 500 오류 (404여야 함) | Jinja2 `UndefinedError` — `watchlist_meta` 미전달 | 예외 핸들러에 누락 컨텍스트 추가 |
| 3 | 관심종목 주가 `—` 표시 | `/api/quotes`가 `?symbols=` 파라미터 무시, 항상 고정 WATCHLIST 조회 | 쿼리 파라미터 처리 추가 |
| 4 | 워치리스트 저장 후 히스토리 미노출 | `briefing_id=None`으로 item insert → 목록 조회에 미포함 | `insert_briefing` 선행 후 item 연결 순서 수정 |
| 5 | 서버 `python3.12-venv` 설치 실패 | Ubuntu 24.04 패키지명은 `python3-venv`, `apt update` 필요 | 패키지명 수정, update 선행 |
| 6 | 배포 후 `.env.example` 서버 미전달 | 숨김 파일이 SFTP 루프에서 자동 제외 | 명시적 업로드 추가 |
| 7 | cron 미등록 → 자동 배치 미실행 | 배포 스크립트에 cron 등록 단계 누락 | paramiko로 cron 등록 + startup 이벤트로 기동 시 자동 실행 |
| 8 | 티커바 오른쪽 카드 화면 밖으로 잘림 | `.ticker-bar`에 `width:100%` 없어 컨테이너 폭 미제한 | `width:100%` · `overflow-y:hidden` · `flex:0 0 auto` · `scroll-snap` 적용 |

**디자인 지시**
- 표 한 장에 8건 전체. 폰트 크기 조정해서 한 화면에 들어오도록
- 슬라이드 오른쪽 상단에 "문제해결 노력 15점" 작은 평가 배지 표시
- 오류 번호 컬럼은 강조색으로

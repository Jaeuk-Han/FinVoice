# DESIGN_BRIEF.md — 해외 금융뉴스 음성 브리핑 서비스

> Claude Design에 붙여넣을 화면 설계 브리프입니다.
> 구현 작업 지시가 아닌, 화면 목적·흐름·규칙을 정리한 참고 문서입니다.

---

## 1. 프로젝트 한 줄 설명

해외 영어 금융뉴스를 수집해 **번역 → 요약 → 음성화**하여 "오늘의 증시 브리핑"을 웹으로 제공하는 서비스.

---

## 2. 주요 사용자

- 로그인 없이 누구나 접속하는 익명 사용자.
- 회원가입·개인화 없음. 모든 사용자가 같은 브리핑을 봄.
- 목표: "오늘 시장/관심종목이 어땠는지" 빠르게 읽거나 듣고 싶음.

---

## 3. 핵심 화면 목록

| # | 경로 | 화면 이름 | 템플릿 |
|---|------|-----------|--------|
| 1 | `/` | 목록 페이지 | `list.html` |
| 2 | `/briefing/{id}` | 상세 페이지 | `detail.html` |
| 3 | `POST /search` | 검색 결과 | `detail.html` 재사용 |

---

## 4. 각 화면의 목적

### 화면 1 — 목록 페이지 (`/`)
- 날짜별 브리핑 목록을 보여줌.
- 상단에 종목 심볼 검색 폼 배치(즉석 검색 진입점).
- 가장 최근(오늘) 브리핑 링크가 첫 번째에 위치.

### 화면 2 — 상세 페이지 (`/briefing/{id}`)
- 하나의 브리핑 날짜에 대한 전체 내용 표시.
- 시장 전반 요약(B) + 종목별 요약 카드(A) 구성.
- 종목 카드: 회사명·심볼·감성 배지·한국어 요약·음성 플레이어·출처 토글.

### 화면 3 — 검색 결과 (`POST /search`)
- 사용자가 종목 심볼을 검색했을 때 즉석(온디맨드) 실행 결과 표시.
- 단일 종목 카드만 노출(상세 페이지 템플릿 재사용).
- 당일 캐시가 있으면 즉시 반환, 없으면 파이프라인 즉석 실행 후 반환.

---

## 5. 입력값과 결과값

### 입력
- **종목 심볼** 텍스트 1개 (예: `AAPL`, `TSLA`).
- 대문자 자동 정규화.
- 지원 종목 힌트 표시 (placeholder 또는 안내 문구).
- 제출 버튼 1개. 그 외 입력 필드 없음.

**지원 종목 (고정)**: AAPL, TSLA, NVDA, MSFT, AMZN, GOOGL

### 결과
- 날짜 + 종목명(회사명 + 심볼)
- 감성 배지: `긍정(positive)` / `중립(neutral)` / `부정(negative)`
- 한국어 요약문
- 음성 플레이어 (mp3가 있을 때만 표시)
- 출처 링크 (접이식 `<details>`)
- 푸터 면책 문구: "투자 조언이 아닙니다"

---

## 6. 3분 발표 최소 흐름

```
[목록 페이지 진입]
       ↓
[검색 폼에 "TSLA" 입력 → 검색 버튼 클릭]
       ↓
[로딩 상태: 스피너 + "요약 생성 중…" 메시지, 버튼 비활성]
       ↓
[검색 결과 화면: 종목 카드 표시 (감성 배지 + 요약 + 음성 플레이어)]
       ↓
[음성 재생 버튼 클릭 → 브라우저 내 재생]
       ↓
[다시 시도: 검색 폼으로 돌아가 다른 종목 입력 (예: "AAPL")]
```

발표에서 보여줄 핵심: **검색 → 로딩 → 카드 → 재생** 4단계.

---

## 7. 로딩 / 오류 / 빈 상태

### 로딩
- 온디맨드 검색은 파이프라인 즉석 실행으로 수 초 소요.
- 진행 중 표시: 스피너 + "요약 생성 중…" 안내 문구.
- 제출 버튼 비활성화(중복 요청 방지).

### 오류
| 상황 | 메시지 (한국어) |
|------|----------------|
| 미지원 종목 입력 | "지원하지 않는 종목입니다. AAPL, TSLA 등 지원 종목을 입력해주세요." |
| 브리핑 없음 | "아직 브리핑이 없습니다. 잠시 후 다시 시도해주세요." |
| 파이프라인 실패 | "요약 생성에 실패했습니다. 잠시 후 다시 시도해주세요." |

기술적 스택트레이스·에러 코드는 화면에 절대 노출하지 않음.

### 빈 상태
- 브리핑 목록이 없을 때: "아직 브리핑이 없습니다." + 검색 폼으로 유도.
- 검색 결과가 없을 때: 빈 화면 대신 안내 문구 + 다른 종목 검색 유도.

---

## 8. 화면 밀도

**실제 앱처럼 보이되, 군더더기는 없게.**

- 단일 컬럼 레이아웃 (`max-width: 760px`), system-ui 폰트.
- 카드형 종목 블록: 옅은 회색 보더 + 8px 라운드.
- 화려한 장식·그라데이션·불필요한 아이콘 남발 금지.
- 충분한 명도 대비 유지 (옅은 회색 본문 텍스트 금지).
- 발표용 단순화가 아닌, 실제 서비스 품질로 제작.

**감성 색상 규칙 (변경 금지)**
- `.s-positive` → `#0a0` (긍정)
- `.s-negative` → `#c00` (부정)
- `.s-neutral` → `#888` (중립)

---

## 9. 버튼 우선순위

1. **검색(제출) 버튼** — 가장 먼저, 가장 크게. 종목 심볼 입력 후 즉시 누를 수 있어야 함.
2. **음성 재생 버튼** — 결과 카드에서 두 번째 핵심 행동.
3. 출처 펼치기 토글 — 부가 정보, 우선순위 낮음.
4. 날짜별 브리핑 목록 링크 — 보조 탐색.

---

## 10. 모바일에서 가장 중요한 행동

1. **검색 폼 입력 + 제출** — 단일 컬럼에서 터치하기 쉬운 크기와 간격.
2. **음성 재생 버튼** — 화면 하단에서도 닿기 쉽도록 배치.

모바일 우선 고려 사항:
- 버튼 최소 터치 크기 확보.
- 결과 카드에서 스크롤 없이 핵심 정보(감성 배지 + 요약 첫 줄 + 재생 버튼)가 보여야 함.
- 외부 링크는 `target="_blank" rel="noopener noreferrer"` 필수.

---

## 11. 더미 데이터 규칙

- 종목 심볼: `AAPL`, `TSLA`, `NVDA`, `MSFT`, `AMZN`, `GOOGL` — 실제 심볼 사용 가능.
- 회사명: Apple Inc., Tesla Inc. 등 공식명 사용 가능 (공개 정보).
- 요약 텍스트: "애플은 이번 분기 시장 예상을 상회하는 실적을 발표했습니다." 형태의 명백한 예시 문장.
- 날짜: `2026-06-06` 형태의 오늘 날짜 또는 가상 날짜.
- 음성 파일: 실제 mp3 없을 경우 플레이어 UI만 표시(src 비움).
- 출처 링크: 실제처럼 보이는 더미 URL 사용 금지 → `https://example.com/news/1` 형태.

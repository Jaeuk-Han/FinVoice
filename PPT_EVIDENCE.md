## 2. 대상 사용자와 문제

> **"영어 뉴스 장벽 없이, 매일 아침 내 관심 종목 브리핑을 30초 만에"**

### 기획 배경
AI 열풍이 반도체·메모리 업종의 주가 호황을 이끌면서, 엔비디아(NVDA)·마이크론 등 미국 테크 종목에 대한 국내 투자자의 관심이 급증했다. 하지만 핵심 뉴스는 대부분 영어로 제공되어 정보 접근에 언어 장벽이 존재한다. FinVoice는 이 격차를 해소하기 위해 기획됐다.

### 해결하려는 문제
- 영어 뉴스를 빠르게 소화하지 못하면 정보가 뒤처져 고점 매수·저점 매도 위험이 커짐
- 시간이 없거나 영어가 불편한 국내 투자자는 동일 정보에 동시 접근이 어려움
- "오늘 내 관심 종목이 왜 올랐는지/내렸는지"를 빠르게 파악할 수단이 부족

### 대상 사용자
| 유형 | 설명 | 제공 기능 |
|------|------|-----------|
| **비로그인 사용자** | 빠르게 오늘 시장 동향을 파악하고 싶은 누구나 | 고정 관심종목(AAPL·TSLA·NVDA·MSFT·AMZN·GOOGL) 브리핑 열람·검색 |
| **로그인 사용자** | 자신만의 포트폴리오를 추적하고 싶은 개인 투자자 | 관심종목 직접 설정(최대 5개), Finnhub에 존재하는 임의 종목 검색 허용 |

> 고정 종목은 AI·반도체 호황의 핵심 미국 빅테크·반도체 종목(NVDA·MSFT·AAPL 등)으로 구성했다.

### 기대 효과
- 언어 장벽 없이 국내 투자자도 동일한 정보에 동시 접근 가능
- 영어 뉴스를 읽지 않아도 30초 안에 오늘 주요 종목 동향 파악
- 출퇴근 중 음성으로 청취 가능
- 개인화된 종목 트래킹 가능

> ⚠️ 본 서비스는 정보 제공 목적이며 투자 조언을 제공하지 않습니다.

## 3. 실제 동작하는 MVP 주요 기능 흐름

### 핵심 파이프라인 (4단계)
```
Finnhub API (영어 뉴스 수집)
    ↓
NCP Papago (영→한 번역)
    ↓
NCP CLOVA Studio HCX-003 (한국어 요약 + 감성 분석)
    ↓
NCP CLOVA Voice (TTS → mp3)  +  NCP Object Storage (mp3 저장)
    ↓
웹 화면 (요약 텍스트 + 음성 플레이어 + 출처 링크)
```

### 브리핑 생성 진입점 3가지 (동일 파이프라인 공유)
| 진입점 | 시점 | 대상 |
|--------|------|------|
| **cron 배치** | 매일 오전 7시 자동 | 고정 WATCHLIST 6개 종목 |
| **서비스 기동 시** | 서버 재시작 직후 (오늘 브리핑 없을 때만) | 고정 WATCHLIST 6개 종목 |
| **온디맨드 검색** | 사용자가 종목 검색 시 | 요청한 종목 1개 |

### 사용자 흐름 (로그인)
```
회원가입 → 로그인 → 관심종목 편집(칩 UI) → 저장
    → 백그라운드 자동 브리핑 생성
    → 3초 폴링으로 완료 감지 → 자동 새로고침
    → 홈에서 개인화 티커바(실시간 주가) + 브리핑 히스토리 확인
```

### 실시간 주가 티커바
- 상단 티커바에 관심종목 실시간 주가 표시 (`▲/▼ + 등락률`)
- 로그인 사용자: 개인 워치리스트 심볼 / 비로그인: 고정 WATCHLIST
- 서버 사이드 60초 캐시로 Finnhub API 요청 최소화

## 4. AI 기능의 입력과 출력

### 4-1. NCP Papago (번역)
| 구분 | 내용 |
|------|------|
| **입력** | Finnhub에서 수집한 영어 기사 제목 + 본문 발췌 (최대 4,500자/요청) |
| **출력** | 한국어 번역문 |
| **사용 위치** | `pipeline/translate.py` |
| **에러 처리** | 번역 실패 시 영어 원문으로 요약 단계 진행 (부분 성공 허용) |

### 4-2. NCP CLOVA Studio — HyperCLOVA X (요약 + 감성 분석)
| 구분 | 내용 |
|------|------|
| **입력** | 종목명 + 번역된 기사 **최대 5건** 목록 (한국어 제목·본문) — 단일 기사보다 다양한 맥락 종합 가능 |
| **프롬프트 전략** | "기사 내용에서만 근거하라. 추측하지 마라. 3~5문장 요약 + positive/neutral/negative 판단. JSON으로만 답하라." |
| **출력** | `{"summary": "한국어 요약문", "sentiment": "positive"}` → 화면에서 감성 배지(초록/빨강/회색)로 시각화 |
| **모델** | HCX-003 (`maxTokens: 500, temperature: 0.3`) |
| **사용 위치** | `pipeline/summarize.py` |
| **에러 처리** | 요약 실패 시 해당 종목 item 생성 건너뜀 (부분 성공 허용, 전체 배치 중단 없음) |

**실제 프롬프트 예시:**
```
다음은 'Apple' 관련 최신 뉴스 기사들이다.
- 애플, 신형 맥북 출시 발표: 애플이 M4 칩 탑재 맥북을 ...
...
핵심 이슈를 한국어 3~5문장으로 요약하고, 전반적 시장 감성을
positive/neutral/negative 중 하나로 판단하라.
반드시 JSON 형식으로만 답하라: {"summary": "...", "sentiment": "..."}
```

**실제 출력 예시 (NVDA):**
```json
{
  "summary": "엔비디아가 AI 데이터센터 수요 급증으로 분기 매출 전망을 상향 조정했다. 블랙웰 GPU 공급 확대와 함께 주요 클라우드 기업들의 대규모 주문이 이어지고 있으며, 경쟁사 대비 AI 가속기 시장 점유율이 더욱 확대될 것으로 전망된다. 단기적으로는 수출 규제 리스크가 남아 있으나 전반적인 펀더멘털은 긍정적이다.",
  "sentiment": "positive"
}
```

### 4-3. NCP CLOVA Voice (TTS)
| 구분 | 내용 |
|------|------|
| **입력** | 한국어 요약문 (최대 1,900자) |
| **출력** | mp3 바이너리 → NCP Object Storage 업로드 → 공개 재생 URL |
| **화자** | `nara` (여성 음성) |
| **사용 위치** | `pipeline/tts.py` |
| **에러 처리** | TTS 실패 시 `audio_url=None`, 텍스트 요약만 표시 (비치명적) |

## 5. 사용한 Ncloud 서비스와 이유

| 서비스 | 역할 | 선택 이유 |
|--------|------|-----------|
| **Server (Ubuntu VM)** | FastAPI 앱 + cron 배치 호스팅 | 안정적인 24/7 서비스 운영, systemd 자동 복구 |
| **Cloud DB for MySQL** | 브리핑·사용자·워치리스트 저장 | 관리형 DB로 백업·이중화 부담 없음 |
| **Papago NMT** | 영→한 번역 | NCP 통합 인증, 금융 도메인 번역 품질 |
| **CLOVA Studio (HyperCLOVA X)** | 핵심 요약 + 감성 분석 | 한국어 금융 문맥 이해도, NCP 내 통합 |
| **CLOVA Voice** | 한국어 TTS mp3 생성 | 자연스러운 한국어 음성, NCP 통합 인증 공유 |
| **Object Storage (S3 호환)** | 생성된 mp3 저장·서빙 | 영구 URL, boto3 표준 인터페이스, 저렴한 스토리지 비용 |

**NCP 인증 구조:** `NCP_APIGW_KEY_ID / NCP_APIGW_KEY` 하나로 Papago·CLOVA Voice 공통 인증 → 키 관리 단순화. Object Storage는 mp3 업로드 시 `public-read ACL` 적용 → 별도 서명 없이 재생 URL 직접 서빙 가능.

**외부 서비스 (NCP 아님)**
| 서비스 | 역할 | 선택 이유 |
|--------|------|-----------|
| **Finnhub API** | 영어 금융뉴스 수집, 실시간 주가, 종목 회사명 조회 | NCP에 영어 금융뉴스 수집 서비스 없음, 무료 티어로 충분 |
| **Clearbit Logo** | 종목 로고 이미지 | 프론트 직접 호출, 무인증, CDN 서빙 |

## 6. 데모 화면 목록

| 화면 | URL | 주요 포인트 |
|------|-----|------------|
| **홈 (브리핑 목록)** | `/` | 날짜별 이력, 검색 폼, 실시간 주가 티커바, 로그인 상태 표시 |
| **브리핑 상세** | `/briefing/{id}` | 감성 배지(positive/neutral/negative), 한국어 요약, 음성 플레이어, 출처 접이식 링크 |
| **온디맨드 검색 결과** | `POST /search` | 검색 후 즉석 AI 파이프라인 실행 결과 표시 |
| **로그인** | `/login` | 이메일·비밀번호, 오류 인라인 표시 |
| **회원가입** | `/register` | 이메일·비밀번호, 중복 이메일 처리 |
| **관심종목 편집** | `/watchlist/edit` | 칩 UI, 최대 5개 제한, 저장 후 자동 브리핑 생성 |

**감성 배지 색상 규칙**
| 감성 | 색상 | 헥스 |
|------|------|------|
| positive | 초록 | `#00c471` |
| negative | 빨강 | `#f04452` |
| neutral | 회색 | `#8c919e` |

**데모 추천 흐름:**
1. 홈 접속 → 오늘 브리핑 확인 (티커바 주가 포함)
2. AAPL 클릭 → 상세 화면 (요약·음성 재생·감성 배지)
3. 회원가입 → 로그인 → 관심종목 편집 (META 추가)
4. 저장 후 자동 새로고침 → META 브리핑 생성 확인
5. (선택) 온디맨드 검색 중 로딩 스피너·버튼 비활성 상태 시연

- Object Storage mp3 업로드 확인: `audio/{date}/AAPL.mp3` 실제 재생 URL 서빙 (TTS 실패 시 텍스트 요약만 표시)
- `pytest -q` 실행: 외부 API 전체 mock 처리, 파이프라인 단위 테스트 포함

### 개발 중 발생한 주요 오류와 해결 기록

| # | 오류 | 원인 | 해결 |
|---|------|------|------|
| 1 | `AttributeError: module 'bcrypt' has no attribute '__about__'` | bcrypt 4.x/5.x가 passlib 1.7.4와 비호환 | `requirements.txt`에 `bcrypt==3.2.2` 버전 고정 |
| 2 | 예외 핸들러 500 오류 (원래 404여야 함) | `html_exception_handler`가 `watchlist_meta` 미전달로 Jinja2 `UndefinedError` | 예외 핸들러에 누락 컨텍스트 모두 전달 |
| 3 | 관심종목 커스텀 주가 `—` 표시 | `/api/quotes`가 항상 고정 WATCHLIST만 조회, `?symbols=` 파라미터 무시 | 엔드포인트에 `symbols` 쿼리 파라미터 처리 추가 |
| 4 | 워치리스트 저장 후 히스토리 미노출 | 백그라운드 배치에서 `briefing_id=None`으로 insert하여 목록 조회에 안 잡힘 | `insert_briefing` 먼저 실행 후 `briefing_item` 연결 |
| 5 | 서버 `apt-get install python3.12-venv` 실패 | Ubuntu 24.04 패키지명은 `python3-venv`, `apt-get update` 필요 | 패키지명 수정, update 선행 |
| 6 | 배포 후 `.env.example` 서버 미전달 | 숨김 파일(`.`으로 시작)은 SFTP 루프에서 자동 제외 | `sftp.put('.env.example', ...)` 명시적 업로드 추가 |
| 7 | cron 미등록 → 자동 배치 미실행 | 배포 스크립트에 cron 등록 단계 누락 | paramiko로 cron 등록 추가, startup 이벤트로 서비스 기동 시 자동 실행 |
| 8 | 티커바 오른쪽 카드 화면 밖으로 잘림 | `.ticker-bar`에 `width: 100%` 없어 `max-width`만으로 컨테이너 폭 미제한 | `width: 100%`, `overflow-y: hidden`, `flex: 0 0 auto`, `scroll-snap` 적용 |

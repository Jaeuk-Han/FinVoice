## 2. 핸드오프 분류

### 그대로 쓸 것
| 대상 | 근거 |
|------|------|
| `app.css` 전체 CSS 변수 (`--ink`, `--s-positive/negative/neutral` 등) | 브리프 §8 감성 색상 규칙과 일치, 변경 금지 |
| `.badge`, `.s-positive/negative/neutral` 클래스 | CLAUDE.md §5 색상 규칙과 동일 |
| `.card`, `.zone`, `.zone-label` 카드 구조 | 브리프 §8 카드형 블록 스펙과 일치 |
| `.player`, `.play-btn` 음성 플레이어 UI | 터치 크기·시각 계층 기준 충족 |
| `.center-state`, `.spinner`, `.load-step` 로딩 UI | 브리프 §7 로딩 스펙과 일치 |
| `.disclaimer` 면책 푸터 | CLAUDE.md §5 "투자 조언 아님" 고정 요건 충족 |
| `.sources`, `.source-list` 출처 접이식 | `rel="noopener noreferrer"` 포함, 브리프 §5 결과 요건 충족 |
| 모바일 반응형 CSS (`.app.is-mobile` 블록) | 제출 버튼 전폭·세로 쌓기·레이아웃 B 단일 컬럼 전환 모두 유효 |

### 고칠 것
| 대상 | 수정 방향 |
|------|-----------|
| `<form id="searchForm">` | `method="POST" action="/search"` 추가, JS preventDefault 제거 → 서버 사이드 처리로 전환 |
| `<audio>` 없이 JS로 시뮬레이션하는 플레이어 | 실제 `<audio src="{{ item.audio_url }}">` 태그로 교체, JS 타이머 제거 |
| JS STOCKS/BRIEFINGS/SOURCES 상수 | Jinja2 `{{ }}` 템플릿 변수로 교체 |
| `fonts/ChosunSg.ttf` 경로 | `app/static/fonts/ChosunSg.ttf`로 경로 확정 및 파일 존재 확인 필요 |
| `max-width: 680px` (.wrap) | 브리프 §8 `max-width: 760px` 기준과 80px 차이 → 프로덕션 적용 전 통일 필요 |

### 버릴 것
| 대상 | 이유 |
|------|------|
| 상단 툴바 전체 (세그먼트 컨트롤·뷰포트·레이아웃 스위처) | 디자인 리뷰용 크롬, 실제 앱에는 불필요 |
| `<div class="device mobile/desktop">` 기기 프레임 | 리뷰 전용 래퍼, Jinja2 템플릿에서 제거 |
| JS 상태 머신 (`S` 객체, `render()`, `viewList()` 등 모든 뷰 함수) | 서버 사이드 렌더링으로 대체 |
| 레이아웃 B·C (분할형·컴팩트형) 카드 변형 | 단일 템플릿에서 레이아웃 선택이 불필요, 레이아웃 A 스택형만 채택 |
| `caption` 영역 ("모든 수치는 데모용 샘플…" 안내문) | 리뷰 전용 설명, 프로덕션 화면에 노출 금지 |

### 다시 요청할 것
| 항목 | 이유 |
|------|------|
| 레이아웃 A 카드에서 오디오 없는 상태 처리 | 현재 음성 플레이어가 항상 렌더됨; TTS 실패 시 플레이어 영역 자체를 숨기는 조건부 UI 디자인 필요 |
| 오류 화면 "브리핑 없음(404)" 전용 메시지 | `viewError`에는 `unsupported`·`pipeline` 2종만 있고, 브리프 §7 "브리핑 없음" 케이스는 `viewEmpty`로만 처리됨 → 분리 여부 확인 |
| `중립(neutral)` 배지 명도 대비 재확인 | `#888` 텍스트는 white 배경 대비 약 3.5:1 (WCAG AA 4.5:1 미달); `--s-neutral` 변경 금지 원칙과의 충돌 여부 판단 필요 |

## 3. 프로덕션 전환 우선순위

1. `app.css` → `app/static/style.css`로 복사 후 CSS 변수·배지·카드·플레이어 클래스 유지
2. 단일 HTML → `list.html` / `detail.html` Jinja2 분리, 폼 action·method 추가
3. JS mock → Jinja2 `{% for %}` 루프 + `{{ }}` 변수 치환
4. `<audio>` 태그 조건부 렌더 (`{% if item.audio_url %}`)
5. 툴바·기기 프레임 제거, 레이아웃 B·C 클래스 정리
